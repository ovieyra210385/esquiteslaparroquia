## 2026-06-23 — Impresión y Vercel

### Build fixes
- Build local fallaba con exit code 1 sin error visible. Causa: caché de Vite corrupto. Solución: borrar `node_modules/.vite` y `dist`, rebuild limpio.
- Vercel deploy fallaba con ENOENT por case-sensitivity. Componentes creados como `paymentqrdialog.tsx` pero importados como `PaymentQRDialog`. Linux es case-sensitive. Se renombraron con `git mv -f` en dos pasos y se commiteó.

### Browser print reescrito
- **Antes (v1):** `window.open('/ticket/print#hash', '_blank')` — falla en tablets por popup blockers
- **Antes (v2):** iframe con `doc.write()` + `iframe.contentWindow.print()` — falla en Android Chrome
- **Ahora (v3):** Overlay div directo en la página con CSS `@media print` que oculta todo excepto el ticket + `window.print()` estándar. Botones "Imprimir" / "Cerrar" visibles. Auto-print a 400ms, auto-cleanup después.
- Función: `printTicketBrowser(data: TicketPrintData)` en `src/lib/utils.ts`
- Se usa en: `pos.tsx` (3 sitios), `configuracion.tsx` (prueba), `ReceiptDialog.tsx` (recibo post-venta)
